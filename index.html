Cosa sono i Thread?

Un thread è la più piccola unità di esecuzione gestita dal sistema operativo. In Java, ogni programma parte con un thread principale (main thread), ma puoi creare thread aggiuntivi per eseguire operazioni in parallelo.

⚡
Concorrenza
Più thread eseguono compiti in modo sovrapposto, condividendo le risorse della CPU tramite il time-slicing del sistema operativo.
🔀
Parallelismo
Su CPU multi-core, thread diversi possono girare letteralmente in parallelo su core fisici distinti allo stesso istante.
📦
Condivisione della Memoria
I thread di uno stesso processo condividono heap e metaspace, ma ognuno ha il proprio stack e program counter privato.
🛡️
Thread Safety
Codice "thread-safe" funziona correttamente anche se eseguito da più thread contemporaneamente, senza race condition o dati corrotti.
Processo vs Thread: Un processo ha il proprio spazio di memoria isolato. I thread sono "fibre" all'interno dello stesso processo: più leggeri da creare, ma richiedono sincronizzazione per accedere a dati condivisi.
Lifecycle

Ciclo di Vita di un Thread

In Java, un thread attraversa diversi stati definiti dall'enum Thread.State.

NEW Creato, non avviato
→
RUNNABLE Pronto / in esecuzione
→
BLOCKED Attende un lock
→
WAITING Attende indefinita
→
TIMED_WAITING Attende con timeout
→
TERMINATED Esecuzione completata
Attenzione: Un thread nel stato TERMINATED non può essere riavviato. Chiamare start() su un thread già terminato lancia un'eccezione IllegalThreadStateException.
Implementazione

Creare un Thread

Ci sono tre modi principali per creare thread in Java.

Metodo 1 — extends Thread 
public class MioThread extends Thread {
 
    @Override
    public void run() {
        // Codice eseguito dal thread
        System.out.println(
            "Thread: " + currentThread().getName()
        );
    }
 
    public static void main(String[] args) {
        MioThread t = new MioThread();
        t.start(); // NON chiamare run()!
    }
}
Metodo 2 — implements Runnable 
public class MioTask implements Runnable {
 
    @Override
    public void run() {
        System.out.println("Runnable in esecuzione!");
    }
}
 
// Utilizzo:
Thread t = new Thread(new MioTask());
t.start();
 
// Oppure con lambda (Java 8+):
Thread t2 = new Thread(() -> {
    System.out.println("Lambda thread!");
});
t2.start();
Metodo 3 — Callable + Future (con valore di ritorno) 
import java.util.concurrent.*;
 
Callable<Integer> task = () -> {
    Thread.sleep(1000);
    return 42;  // Callable può restituire un valore!
};
 
ExecutorService executor = Executors.newSingleThreadExecutor();
Future<Integer> future = executor.submit(task);
 
System.out.println("Aspetto il risultato...");
int result = future.get();  // Blocca fino al completamento
System.out.println("Risultato: " + result);  // 42
 
executor.shutdown();
Best Practice: Preferisci Runnable o Callable all'estensione diretta di Thread. Questo separa il task dalla sua strategia di esecuzione e consente di usare i thread pool.
Concorrenza Sicura

Sincronizzazione

Quando più thread accedono agli stessi dati, si possono verificare race condition. La sincronizzazione garantisce l'accesso esclusivo alle sezioni critiche.

synchronized — Keyword 
public class Contatore {
    private int valore = 0;
 
    // Solo un thread alla volta
    public synchronized void incrementa() {
        valore++;
    }
 
    public synchronized int getValore() {
        return valore;
    }
 
    // Blocco sincronizzato (più flessibile)
    public void incrementaBloccato() {
        synchronized(this) {
            valore++;
        }
    }
}
ReentrantLock — Più controllo 
import java.util.concurrent.locks.*;
 
Lock lock = new ReentrantLock();
int valore = 0;
 
public void incrementa() {
    lock.lock();
    try {
        valore++;
    } finally {
        lock.unlock(); // SEMPRE nel finally!
    }
}
 
// Tentativo non bloccante
if (lock.tryLock()) {
    try {
        // sezione critica
    } finally {
        lock.unlock();
    }
}
Atomic Classes — Lock-free, alte prestazioni 
import java.util.concurrent.atomic.*;
 
AtomicInteger contatore = new AtomicInteger(0);
 
contatore.incrementAndGet();    // ++valore, thread-safe
contatore.decrementAndGet();    // --valore, thread-safe
contatore.addAndGet(5);         // valore += 5
contatore.compareAndSet(10, 20); // se valore==10, imposta a 20
 
// Altre classi atomiche utili:
AtomicLong      longAtomico = new AtomicLong();
AtomicBoolean   flagAtomico = new AtomicBoolean(false);
AtomicReference<String> ref = new AtomicReference<>("valore");

Comparazione Metodi di Sincronizzazione

Thread Pool

ExecutorService e Thread Pool

Creare un thread per ogni task è costoso. I Thread Pool riutilizzano un numero fisso di thread per eseguire molti task in modo efficiente.

Tipi di Thread Pool 
import java.util.concurrent.*;
 
// Pool con numero fisso di thread
ExecutorService fixed = Executors.newFixedThreadPool(4);
 
// Pool che scala dinamicamente (attenzione: può creare molti thread)
ExecutorService cached = Executors.newCachedThreadPool();
 
// Un solo thread, esecuzione sequenziale garantita
ExecutorService single = Executors.newSingleThreadExecutor();
 
// Pool per task pianificati / periodici
ScheduledExecutorService sched = Executors.newScheduledThreadPool(2);
sched.scheduleAtFixedRate(() ->
    System.out.println("Ogni 5 secondi"),
    0, 5, TimeUnit.SECONDS);
 
// Java 21+: Virtual Thread Pool (Project Loom)
ExecutorService virtual = Executors.newVirtualThreadPerTaskExecutor();
Esempio Completo — Elaborazione Parallela 
import java.util.concurrent.*;
import java.util.*;
 
public class ElaborazioneParallela {
    public static void main(String[] args) throws Exception {
 
        ExecutorService pool = Executors.newFixedThreadPool(4);
        List<Future<String>> futures = new ArrayList<>();
 
        // Sottometti 10 task al pool
        for (int i = 0; i < 10; i++) {
            final int taskId = i;
            futures.add(pool.submit(() -> {
                Thread.sleep(500); // simula lavoro
                return "Task " + taskId + " completato";
            }));
        }
 
        // Raccogli i risultati
        for (Future<String> f : futures) {
            System.out.println(f.get());
        }
 
        // Shutdown ordinato: aspetta il completamento
        pool.shutdown();
        pool.awaitTermination(30, TimeUnit.SECONDS);
    }
}
⚠️
Deadlock
Due thread si bloccano a vicenda aspettando un lock che l'altro detiene. Prevenibile acquisendo i lock sempre nello stesso ordine.
🔄
Livelock
I thread non sono bloccati ma continuano a cambiarsi di stato reciprocamente senza progredire. Difficile da individuare.
🏃
Starvation
Un thread non ottiene mai il lock perché altri thread a priorità più alta lo acquisiscono continuamente prima di lui.
Novità Java 21+

Virtual Threads (Project Loom)

I Virtual Thread sono thread leggeri gestiti dalla JVM (non dall'OS). Puoi crearne milioni senza esaurire la memoria. Ideali per applicazioni I/O-intensive (server web, database). Basta cambiare una riga di codice!
Virtual Threads — Java 21 
// Prima (Platform Thread)
Thread t = new Thread(() -> task());
 
// Dopo (Virtual Thread) — stessa API!
Thread vt = Thread.ofVirtual().start(() -> task());
 
// Con ExecutorService
try (ExecutorService exec = Executors.newVirtualThreadPerTaskExecutor()) {
    for (int i = 0; i < 1_000_000; i++) {  // 1 MILIONE di task!
        exec.submit(() -> {
            Thread.sleep(Duration.ofMillis(100));
            return "OK";
        });
    }
} // auto-shutdown con try-with-resources
